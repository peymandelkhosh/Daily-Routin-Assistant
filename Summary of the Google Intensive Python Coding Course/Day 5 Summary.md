# Deploy an ADK agent to Agent Runtime using Agents CLI

> **یادداشت‌های روز پنجم دوره فشرده توسعه هوش مصنوعی گوگل**

---

## مرحله ۱: مقدمه و اهداف کلی (Overview)

### خلاصه درس
در این بخش، هدف انتقال یک ایجنت توسعه‌یافته با کیت توسعه ایجنت گوگل (ADK 2.0) از محیط توسعه محلی (Localhost) به یک محیط ابری آماده تولید و پایدار به نام **Agent Runtime** در گوگل کلود (Google Cloud) است. ابزار خط فرمان اصلی مورد استفاده در این بخش `agents-cli` نام دارد.

به عنوان سناریوی عملی، ایجنت هزینه (Ambient Expense Agent) را بررسی می‌کنیم. این ایجنت فاکتورهای کاری کارمندان را اسکن کرده و طبق قوانین زیر تصمیم‌گیری می‌کند:
1. **تایید خودکار (Auto-Approve):** فاکتورهای با مبلغ زیر ۱۰۰ دلار فوراً تایید می‌شوند.
2. **بررسی انسانی (Human-in-the-Loop):** فاکتورهای با مبلغ ۱۰۰ دلار یا بیشتر متوقف شده و نیاز به تایید مدیر دارند.

### اهداف یادگیری
* آماده‌سازی پروژه محلی ایجنت برای انتقال به ابر گوگل.
* ساخت فایل‌های توصیف‌گر استقرار و کدهای مربوط به وب‌سرویس تولید.
* تست محلی و پیش‌نمایش استقرار (Dry-run) برای شناسایی مشکلات احتمالی.
* استقرار مستقیم کدهای ایجنت به Agent Runtime.
* پایش و رهگیری فرآیندهای ایجنت در محیط ابری با سرویس Cloud Trace.

### پیش‌نیازهای مورد نیاز
* یک پروژه فعال در Google Cloud با بخش پرداخت فعال (Billing Enabled).
* کیت توسعه gcloud SDK نصب و لاگین‌شده روی سیستم محلی.
* ابزار مدیریت بسته پایتون uv.
* محیط توسعه Google Antigravity IDE.

## مرحله ۲: پیکربندی محیط گوگل کلود (GCP Setup)

قبل از استقرار ایجنت، لازم است متغیرهای ابری تنظیم شده و دسترسی‌های زیر (API) در پروژه گوگل کلود فعال شوند:
* `aiplatform.googleapis.com` (سرویس اصلی هوش مصنوعی ورتکس)
* `cloudtrace.googleapis.com` (سرویس پایش و مانیتورینگ عملکرد)
* `cloudbuild.googleapis.com` (سرویس ساخت خودکار برنامه)
* `agentregistry.googleapis.com` (رجیستری ایجنت‌های سازمانی)
